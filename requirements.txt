# L'extra [crypto] installe le paquet "cryptography", requis par pypdf pour
# le chiffrement/dechiffrement AES-256 (protection par mot de passe). Sans
# lui, set_password()/remove_password() echouent avec DependencyError.
#
# Borne haute <7.0 : pdf_ops.py documente explicitement (reorder_and_filter_
# pages, add_text_watermark) que pypdf 7.0 va deprecier le fait de modifier
# une page encore rattachee a un reader seul, un comportement dont depend le
# code actuel. Sans cette borne, un futur `pip install -r requirements.txt`
# installerait 7.0 silencieusement, sans que le code n'ait ete teste/adapte
# contre ce changement (bombe a retardement trouvee a l'audit) - a lever
# uniquement apres avoir valide la suite de tests contre pypdf 7.0.
pypdf[crypto]>=6.0,<7.0
pypdfium2>=5.0
reportlab>=4.0
Pillow>=10.0

tkinterdnd2>=0.6.2

# build local - constat de l'audit). Un venv dedie garantit un executable
# plus petit et reproductible a partir de ce seul fichier :
